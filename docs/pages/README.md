# Customer-Facing Pages

This folder contains Markdown source copy for the public/customer-facing Fastifly website.

These files are not implementation specs. They translate the implementation foundation into language a user can understand before they install or use the product.

## Page Map

| Page | Customer question answered | Implementation source |
|---|---|---|
| `home.md` | What is Fastifly and why should I care? | Phases 0-5 combined |
| `reliable-finance-foundation.md` | Why should I trust this app with financial records? | Phases 0-3 |
| `self-hosting-and-data-ownership.md` | Where does my data live and how do I run it? | Phases 0, 2, 3 |
| `sign-in-and-recovery.md` | How do I access my account safely? | Phase 4 |
| `workspaces-and-permissions.md` | Can I share finance data without losing control? | Phase 4 |
| `safe-saving-and-retries.md` | What happens if I click twice, go offline, or retry? | Phase 5 |

## Writing Rules

- Write for users, not maintainers.
- Do not mention package names, table names, test commands, or internal class names unless the page is explicitly for technical buyers.
- Be honest about current scope.
- Avoid promising bank sync, mobile apps, AI features, or full offline editing until they exist.
- Prefer concrete outcomes: fewer duplicate transactions, safer retries, clear roles, recoverable access, self-owned data.
